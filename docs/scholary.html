Project abstract

Requirements

Develop a Web tool (framework) using natural language constructs to interact to a REST API. The tool will (re)use knowledge model(s) to learn the proper concepts exposed by the OpenAPI document describing this API (e.g., resources, their inputs and outputs) to facilitate a proper text/voice-based interaction. Test and demonstrate the system's features by choosing minimum two public REST APIs. Inspiration: Photon (article: A Robust Cross-Domain Text-to-SQL System). Bonus: provide multi-lingual support.

Proposed solution

We propose a multi-service application which will aggregate a multitude of micro-service and will provide a user-friendly interface.

The interface should be used to input a link of an OpenAPI and a sentence that describes the wanted interaction with the specified API.

We will have a central API point that will play the role of the static delivery hub and of a gateway for the rest of the services. The services we will use are:

  • Translation Service

    Will take the natural language query and will translate it to english. English is the language used in most OpenAPi descriptions.

  • Preprocessing Service

    Will interrogate the OpenAPI and process the response in order to extract the keywords. This service will attempt to create a mapping from english words to API routes.

  • NaturalLanguage Service

    Will use the information provided by Translation Service and Preprocessing Service in order to compute a query. We will try to use the knowledge we have in order to create an API call and return it to the user.

Implementation details

Programming languages

Python

We chose Python for it's easy to use and plentiful Machine Learning libraries.

JavaScript

We chose JavaScript for the front-end part of the application. We intend to use a JS framework called Angular in order to create a responsive and modern user interface.

Typescript

Additionally, for the services that do not require additional processing libraries, we opted for Typescript for all the JavaScript benefits but with the addition of readability and reliability provided by the support for types.

Architecture

The application of our application can be seen in the image below.

Angular Frontend application

The user will interact with an Angular front-end application where he/she can specify the link to an OpenAPI document and the sentence that describes the wanted interaction with the specified API.

API Gateway

The front-end application will send the request to an orchestration service. This service, contained in a Typescript Azure Function, is responsible for managing the execution steps of our proposed solution.

Translation Service

If the translation of the sentence is required, it will send a request to the translation service, also a Typescript Azure Function, that will directly interact with the Google Translate API and translate the sentence.

Preprocessing and Natural Language Processing services

After translation, the orchestration service interacts with the preprocessing service and then the processing service in order to transform the input into the wanted request, both Python Azure Functions.
